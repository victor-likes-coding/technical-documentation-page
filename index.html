Variables

One of the very first things you must know about JavaScript is that it uses variables like most other languages. When thinking about variables, think of a bucket, or a box. The bucket or box will contain something inside of it. You name the bucket/box so that you can then reference it later when you need what is in the container. Look below:

const cookie = 5;

In that statement, or expression, I've set the word cookie to have a value of 5. In English, you'd say something like, set the word cookie to equal 5. Now when I want to use it, I call cookie. Calling it is simply typing the word like so: cookie * 5. What's happening is JavaScript won't know what cookie is, so it goes up a chain and tries to find some meaning. It ends up understanding that cookie is 5 so the expression goes from cookie * 5 to 5 * 5. It might not make a whole lot of sense now, but variables are very important.

Data Types

In the previous section, we talked about variables. Variables are just words we use to store some value. Variables can hold all sorts of things, but they're categorized into distinct types. Some of those types are in JavaScript:

  • Strings
  • Number
  • Boolean
  • undefined
  • BigInt
  • Symbols

Strings are just letters and characters that goes between quotes, single or double. "A" is the same as 'A'. They're literals, meaning what you put in between the quotes is literally what you'll see (most of the time).

Numbers involve strictly numbers. Floats such as decimal numbers, and integers such as big rounded numbers without decimals. Some examples of integers: 11 50 100 999, and floats: 1.11 1.34 55.51

Booleans are either true or false. In JavaScript, they're represented as true/false.

To the extent that I will do the minimal amount of work to get this project done, I will skip undefined, BigInt, and Symbols.

Conditional Statements

A conditional statement is a set of commands that executes if a specified condition is true. JavaScript supports two conditional statements: if...else and switch.

An if statement looks like this:

if (condition) {
 statement_1;
} else {
 statement_2;
}

Here, the condition can be any expression that evaluates to true or false. If condition evaluates to true, statement_1 is executed. Otherwise, statement_2 is executed. statement_1 and statement_2 can be any statement, including further nested if statements.

Loops

Loops offer a quick and easy way to do something repeatedly. You can think of a loop as a computerized version of the game where you tell someone to take X steps in one direction, then Y steps in another. For example, the idea "Go five steps to the east" could be expressed this way as a loop:

for (let step = 0; step < 5; step++) {
 // Runs 5 times, with values of step 0 through 4.
 console.log('Walking east one step');
}
